# BeamNG Real-Life Map Generator

Генератор карт для **BeamNG.drive** из данных реального мира:
- OSM (формы, дороги, объекты);
- AWS Terrain workflow (загрузка/подготовка heightmap-пайплайна);
- автоматическая генерация текстур;
- добавление road nodes;
- экспорт готового мода (`.zip`) в папку `Downloads`.

## Стек
- Backend + desktop launcher: Rust (`axum` + `tokio`)
- Интерфейс: TypeScript + React + Leaflet (Vite)
- CI: GitHub Actions (автосборка portable `.exe` + UI assets)

## Как работает portable .exe
- Запускаете `beamng-map-generator.exe`.
- Приложение поднимает локальный сервер и **автоматически открывает UI в браузере**.
- На Windows консольное окно отключено (`windows_subsystem = "windows"`).

## Локальный запуск для разработки
```bash
cd ui
npm install
npm run build
cd ..
cargo run
```

По умолчанию приложение стартует на `http://127.0.0.1:8080`.

Переменная порта:
```bash
BEAMNG_MAP_GENERATOR_PORT=8080 cargo run
```

Если порт занят, сервер автоматически попробует свободный порт в диапазоне `+1..+20` и откроет корректный URL в браузере.

## Важные изменения UI
- Выбор области делается через Leaflet-режим выделения: включаете режим и выделяете прямоугольник мышью на карте.
- Ручного ввода `north/south/east/west` в форме нет.
- Добавлен второй режим источника данных: **локальный `.osm` файл** (кнопка «Экспорт» на openstreetmap.org).

## OSM и устойчивость загрузки
- Онлайн-режим использует несколько Overpass endpoint'ов (fallback), а не только один сервер.
- Если у конкретного пользователя основной Overpass-инстанс недоступен/перегружен, автоматически используются зеркала.
- Поддержан импорт локального `.osm` файла: API `POST /api/generate-from-osm`.

## Примечания
- Heightmap модуль в `src/aws_terrain.rs` сделан как интеграционный слой под AWS Terrain/S3 DEM пайплайн и сейчас содержит детерминированный генератор-заглушку для локальной разработки.
- Готовый архив мода сохраняется в системную папку `Downloads` (или в рабочую директорию, если путь недоступен).


## 3D контент в генерации
- Генерируется `main.materials.json` с terrain и building материалами (albedo/normal/roughness).
- Генерируются procedural PBR-текстуры для стен/крыш зданий.
- Экспортируются реальные road-объекты в `generated_roads.json` (DecalRoad-style JSON).
- Здания экструзируются из OSM `building` footprints в 3D меш `art/shapes/generated/buildings.dae` + prefab `generated_buildings.prefab.json`.
- Heightmap формируется из AWS Terrain (Terrarium tiles `elevation-tiles-prod`).


## Текстуры, трафик, спутник, деревья, коллизии
- Качество текстур terrain = `texture_resolution` из UI (до 8192), поэтому можно генерировать чёткие карты.
- Дороги экспортируются как DecalRoad + отдельно AI path (`generated_ai_paths.json`) для настройки трафика/AI.
- Спутниковый снимок может использоваться как отдельный источник albedo/overlay (полезен для реалистичной подложки), но требует отдельного провайдера и лицензии.
- Лесные зоны OSM (`landuse=forest`, `natural=wood`) конвертируются в prefab инстансы деревьев (`generated_trees.prefab.json`) со стандартной моделью дерева из игры.
- Коллизия: у дорог коллизия штатная через дорожный объект; у зданий коллизия зависит от импорта DAE в BeamNG (обычно нужна финальная проверка/донастройка collision mesh в редакторе).


## Формат мода (совместимость с BeamNG)
Архив теперь собирается в структуре:
- `levels/<map_name>/...` (уровень и его ассеты),
- `mod_info/<ID>/info.json` (метаинформация мода для менеджера модов).

Это ближе к структуре игровых модов (`levels + mod_info`) и корректно распознаётся менеджером модов BeamNG.

## Уровни качества текстур в UI
- 512 — очень низко (неиграбельно)
- 1024 — низко (на грани играбельности)
- 2048 — средне (играбельно)
- 4096 — чётко (комфортно играбельно)
- 8192 — очень чётко (максимально играбельно)
